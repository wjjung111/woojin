# PRD — ARAP 토지건물 V1.0

> 제품요구사항문서 (Product Requirements Document)
> 최초 작성: 2026-07-01 · 상태: 작성중(V1.0 설계)

---

## 1. 한 줄 정의

감정평가사가 **토지·건물 개별평가(토지=공시지가기준법+거래사례비교법, 건물=원가법)** 의견서를 쓸 때,
**평가사 판단 외의 모든 계산·데이터조회·서식작업을 자동화**하는 도구. (집합건물 ARAP의 다음 단계)

## 2. 왜 만드나 (문제)

- 현재 워크플로우: **카파허브에서 복사 → 엑셀에 붙여넣기 → 수식으로 계산**.
- 문제점: (a) 매번 수기 복붙 → 실수·시간소요, (b) 사례 5건 하드코딩 엑셀 → 사례 수 바뀌면 깨짐, (c) 계산결과를 의견서로 다시 옮겨적음.
- 목표: 의견서 1건 **1시간 → 30분 이내** (집합건물 ARAP과 동일한 대원칙).

## 3. 제품의 경계 (가장 중요 — 무엇을 "안" 하나)

집합건물 ARAP의 대원칙을 그대로 계승한다: **"AI가 판단해주는 도구"가 아니라 "판단 외 단순작업 자동화 도구"**.

### ✅ 자동화 대상 (앱이 처리)
- 시점수정 계산 (지가변동률 조회·기간·산식)
- 공시지가기준법 산식 (표준지 × 시점 × 개별요인 × 그밖의요인)
- 거래사례비교법 산식 (건물가격공제법 포함 — 매매가에서 건물값 빼서 토지단가 역산)
- 건물 원가법 (재조달원가 × 잔가율, 경과연수/내용연수 감가)
- 반올림·유효숫자·단가 결정단위 처리
- (V2) 국토부/공공데이터포털 API로 토지특성·공시지가·실거래·지가변동률 자동조회
- (V3) 워드 의견서 템플릿 채우기·다운로드

### ❌ 자동화 비대상 (평가사 수동 유지) — **절대 자동화 시도 금지**
- 표준지 선정 (어떤 표준지를 쓸지)
- 비교 거래사례 선정 (어떤 사례를 쓸지)
- 개별요인 우열 판단 (가로·접근·환경·획지·행정·기타 비교치 결정)
- 그밖의요인 보정치 결정
- 최종 시산가액·결정단가 결정 (공시지가기준법값 vs 거래사례비교법값 중 조정)

> 판단 근거: 엑셀에서 개별요인 비교치(1.02 등)·표준지선정·사례선정은 전부 평가사가 손으로 넣던 값.
> 앱은 "그 값들을 받아서 곱하고 반올림"까지만 한다.

## 4. 타겟 사용자

**현재**: 본인(개발자=사용자, 1인 사무용). 집합건물 ARAP과 동일.

## 5. 대상 물건 (V1.0 우선순위)

1. **단독주택 / 다가구주택** ← 최우선 (가장 많음)
2. **상가주택 / 근린생활시설** ← 그다음
3. (이후) 공장·창고 등 산업용

## 7. 계산 로직 명세 (엑셀 `토지산출(2)` 시트 역설계 결과)

### 7-1. 공시지가기준법
```
산정단가 = 표준지공시지가 × 시점수정치 × 개별요인비교(계) × 그밖의요인보정치
결정단가 = ROUND(산정단가, -4)          // 만원 단위 반올림
시산가액 = 결정단가 × 본건 토지면적
```
- 개별요인비교(계) = ROUND( 가로 × 접근 × 환경 × 획지 × 행정 × 기타, 3 )
- 시점수정치 = 지가변동률 월별수치의 누적곱 (부분월은 일할계산)
  - 예: `(1 + 0.00471) × (1 + 0.00490 × 25/28) ≒ 1.00911`

### 7-2. 거래사례비교법 (건물가격공제법)
```
[사례별]
건물평가액 = 건물재조달원가단가 × 잔가율 × 건물면적
  · 잔가율 = (내용연수 - 경과연수) / 내용연수
  · 경과연수 = 기준시점 - 사용승인일 (연)
토지가액   = 매매금액 - 건물평가액
토지단가   = 토지가액 / 토지면적
시점보정단가 = 토지단가 × 시점수정치

[적용]
산정단가 = 시점보정단가 × 개별요인비교(본건/사례)
결정단가 = ROUND(산정단가, -4)
시산가액 = 결정단가 × 본건 토지면적
```

### 7-3. 건물 원가법
```
건물시산가액 = 재조달원가 × 잔가율 × 건물면적   // 세부는 건물산출 시트 확인 후 확정(V1.0 Sprint3)
```

### 7-4. 통합
```
최종 = 토지 시산가액(평가사 결정) + 건물 시산가액
```
> 토지는 공시지가기준법·거래사례비교법 두 값을 나란히 보여주고 **격차율**(두 값 차이 비율)을 표시.
> 최종 결정은 평가사가.

## 9. 기술 방향

- **단일 HTML 파일, 빌드도구 없음** — 브라우저에서 바로 실행 (집합건물 ARAP과 동일 철학).
- V1.0은 **바닐라 JS**(계산기라 React 불필요·가볍고 읽기 쉬움). 초심자 학습·유지보수 용이.
- 서버 없음. API 연동(V2) 시 CORS 문제는 그때 별도 설계(프록시 or 데스크톱 로컬서버).
