# 의견서 v2.0 (새 감정평가서 양식) 플레이스홀더 매핑

- 템플릿 파일: `템플릿/exact v2.0 감정평가서 템플릿_주거용+단일호수.hwpx`
- 원본: 사용자 제공 감정평가서 한글 양식(주거용·단일호수 샘플, 표지+감정평가표+산출근거)을 hwpx로 저장한 것
- 플레이스홀더 총 **125종**, `{{토큰}}` 형식 (기존 v1.2 DOCX와 동일한 치환 방식 — `buildHwpx` 계열 엔진으로 처리 가능)
- 샘플값 예시는 원본에 들어있던 값 (전은양 / 정든마을 506동 702호 건)

## 섹션 구성
| 파일 | 내용 |
|---|---|
| `Contents/section0.xml` | 표지 + (구분건물)감정평가표 |
| `Contents/section1.xml` | 감정평가액의 산출근거 및 결정의견 (의견서 본문) |

## 주의사항 (템플릿 생성 시 처리한 것)
- 한글 표 **계산식(FORMULA) 필드 4개 제거** — 캐시된 샘플값(1,588,000,000 / 126.182)이 남고, 한글에서 재계산되면 플레이스홀더를 덮어쓰기 때문. 해당 칸은 이제 일반 텍스트라 앱이 값을 채워야 함: 감정평가액(감정평가표 2곳), 합계면적.
- 설비 설치유무 9칸(○/-)도 토큰화 — 앱에서 기본값 `○ ○ ○ ○ - ○ ○ - -` 제공 권장.
- 거래사례 후보표는 **6행 고정**(기호 ㉠~㉥ 리터럴). 사례가 6개 미만이면 남는 행 토큰을 빈 문자열로 치환, 7개 이상은 v2.0 엔진에서 행복제 필요(미구현).
