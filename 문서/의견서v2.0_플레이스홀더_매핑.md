# 의견서 v2.0 (새 감정평가서 양식) 플레이스홀더 매핑

- 템플릿 파일: `템플릿/exact v2.0 감정평가서 템플릿_주거용+단일호수.hwpx`
- 원본: 사용자 제공 감정평가서 한글 양식(주거용·단일호수 샘플, 표지+감정평가표+산출근거)을 hwpx로 저장한 것
- 플레이스홀더 총 **129종**, `{{토큰}}` 형식 (기존 v1.2 DOCX와 동일한 치환 방식 — `buildHwpx` 계열 엔진으로 처리 가능)
- **(2차 수정) 사용자 추가 토큰 4종**: `{{감정평가사}}`(서명란), `{{심사자}}`(심사의견 서명란), `{{세대}}`(평가내역 수량, 예: 1세대), `{{유형}}`(물건 유형, 예: 아파트 — 산문 "집합건물({{유형}})" 3곳 + "인근 유사 {{유형}}" 1곳)
- **(2차 수정) 서식 처리**: 모든 플레이스홀더 **파란색**(#0000FF, 전용 charPr 복제본 6종) — 치환 엔진이 값 채운 뒤에도 파란색이 남으므로, v2.0 엔진에서 치환 시 해당 런의 charPrIDRef를 원래 검정 id로 되돌리거나(권장: 파란 charPr id 53~58 → 원본 매핑 유지) 파란 글자모양을 검정으로 바꿔치기해야 함
- **(2차 수정) 겹침 해소**: 토큰이 들어간 문단 141개의 `linesegarray`(줄배치 캐시) 제거 — 원래 짧은 값(○ 등) 기준 캐시 때문에 긴 토큰이 겹쳐 보이던 문제. 한글이 열 때 재계산함
- 샘플값 예시는 원본에 들어있던 값 (전은양 / 정든마을 506동 702호 건)

## 주의사항 (템플릿 생성 시 처리한 것)
- 한글 표 **계산식(FORMULA) 필드 4개 제거** — 캐시된 샘플값(1,588,000,000 / 126.182)이 남고, 한글에서 재계산되면 플레이스홀더를 덮어쓰기 때문. 해당 칸은 이제 일반 텍스트라 앱이 값을 채워야 함: 감정평가액(감정평가표 2곳), 합계면적.
- 설비 설치유무 9칸(○/-)도 토큰화 — 앱에서 기본값 `○ ○ ○ ○ - ○ ○ - -` 제공 권장.
- 거래사례 후보표는 **6행 고정**(기호 ㉠~㉥ 리터럴). 사례가 6개 미만이면 남는 행 토큰을 빈 문자열로 치환, 7개 이상은 v2.0 엔진에서 행복제 필요(미구현).

## v2.0 앱 (완료 — 2026-07-24)
- `exact_집합건물v2.0.html` = v1.0 사본에서 분기 (v1.0 무수정 보존). localStorage 공유라 저장된 건·템플릿 캐시 그대로 이어짐.
- 템플릿 base64 임베드(`V2EVAL_TPL_B64`) + 엔진 `buildEvalHwpxV2`(전 섹션 토큰 치환 + 파란 charPr 53~58 → 검정 복귀 `V2_BLUE2BLACK`) + 매핑 `buildV2TokenMap`(buildData 결과 d를 v2 토큰으로 확장).
- 단일호수 ④ 가격산출 탭에 **[3. 감정평가서(hwp) v2.0]** 버튼(보라색) — 주거용 전용, 비주거용은 안내 후 중단.
- 미매핑 항목: `{{감정평가사}}`·`{{심사자}}`는 `ov.appraiserName`/`ov.reviewerName`(현재 입력칸 없음 → 빈칸 출력, 한글에서 수기), 설비 9칸은 기본값(○○○○-○○--), `{{지수기준월}}`은 상수 "2025.03"(지수 개편 시 코드 수정).
- 검증: 헤드리스 크롬에서 실제 로드 + 압구정동 샘플 데이터로 hwpx 생성 → 미치환 토큰 0, XML 전체 유효, 파란 참조 잔존 0 확인.

## v2.0 2차 수정 (2026-07-24 오후)
- **폰트 대응**: linesegarray(줄배치 캐시) 전역 제거 → **값 셀(토큰만 있는 짧은 문단 132개)만 제거**로 변경. 본문 산문 9문단은 원본 레이아웃 유지(전역 제거가 본문 재배치를 유발한 것으로 추정 — 폰트 자체는 원본과 동일함을 검증). 표 셀 겹침은 값 채워지면 어차피 정상.
- **파란색 유지**: 사용자 요청으로 출력물의 채워진 값도 파란색 유지 → 엔진의 블루→검정 변환(`V2_BLUE2BLACK`) **제거**. 값이 블루 charPr(53~58, 원본과 폰트 동일)로 채워짐. 검증: 채운 값이 charPr 54/55에 들어감 확인.
- **단일호수 버튼바 정리**: 다운로드 바를 **[3. 감정평가서(hwp) v2.0] 보라색 버튼 1개만** 남기고 나머지(의견서 docx·확약서·괄호감정표·명세표) 제거. 표지·청구서·회보서 카드는 별도 섹션이라 유지. **여러호수 탭은 미변경**(v2 템플릿 없음).
